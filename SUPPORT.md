# Support

Thanks for using TokenGauge. Here is how to get help.

## Getting help

- **Read the docs first.** The [README](README.md) covers setup (including the
  Claude Code statusLine bridge and the opt-in Codex probe), configuration, and
  troubleshooting. [PRIVACY.md](PRIVACY.md) and [ACCURACY.md](ACCURACY.md)
  explain exactly what is read and how each metric is labeled.
- **Run the built-in diagnostics.** **TokenGauge: Cockpit Diagnostics** produces
  a bounded, redacted health report. It is the fastest way to see why a card is
  showing unavailable, stale, or disabled.
- **Search existing issues** before opening a new one. Someone may have already
  reported it.
- **Open an issue** using the bug report or feature request template, and please
  follow the privacy note in each template.

## Reporting a security vulnerability

Do **not** open a public issue for a security problem. Follow
[SECURITY.md](SECURITY.md) and report through GitHub Security Advisories.

## Protect your privacy in support requests

TokenGauge is privacy-first, and so are its issues. When asking for help, do not
paste secrets, API keys, raw filesystem paths, raw logs, prompts, completions,
transcripts, terminal output, native status or probe payloads, account IDs,
session IDs, or emails. Redact anything sensitive and use placeholders.

## Supporting the project

TokenGauge is free and open source under Apache-2.0. If it is useful to you, the
most helpful things you can do are star the repository, share it with others who
juggle multiple AI coding agents, and contribute improvements. See
[CONTRIBUTING.md](CONTRIBUTING.md).
